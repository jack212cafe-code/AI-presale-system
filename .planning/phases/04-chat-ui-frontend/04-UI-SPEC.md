# Phase 4 — UI Design Contract

> Visual and interaction contract for the Chat UI Frontend phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none — vanilla HTML/CSS/JS, no framework, no bundler |
| Icon library | none — use Unicode/text labels only |
| Font | "Segoe UI", Tahoma, Geneva, Verdana, sans-serif (source: intake/index.html) |

**Rationale:** Project constraint (CLAUDE.md + RESEARCH.md) — no build step, no framework. CSS variables from `intake/index.html :root` are the design system. Copy the `:root` block verbatim into `chat/chat.html`.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Display / page heading | 28px (1.75rem) | 600 semibold | 1.2 |
| Section heading | 20px (1.25rem) | 600 semibold | 1.2 |
| Body | 16px (1rem) | 400 regular | 1.7 |
| Label / small / muted | 13px (0.81rem) | 400 regular | 1.5 |

Notes:
- Only 2 weights used: 400 (regular) and 600 (semibold) — source: intake/index.html `summary { font-weight: 600 }`
- Muted text (`.message-label`, hints): 13px, color `var(--muted)`, weight 400
- User bubble text: 16px, color `#f7f4ef` on dark background
- UI form labels and nav items: 16px body size (not a separate size step)

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#f6f2ea` (`--bg`) | Page background, sidebar background |
| Secondary (30%) | `rgba(255,255,255,0.84)` (`--surface`) | Assistant bubbles, card surfaces, composer bar |
| Accent (10%) | `#111111` (`--accent`) | User bubble background, primary CTA button, active/selected states |
| Destructive | `#8b2d38` (`--error-ink`) on `#fbefef` (`--error-bg`) | Error banners only |

Accent reserved for:
1. User message bubble background (`#111111` fill)
2. Primary send button (`background: var(--accent)`)
3. "เลือกตัวเลือกนี้" solution selection button (same primary button style)
4. Active/focused input border emphasis (`rgba(17,17,17,0.26)` border + `rgba(17,17,17,0.06)` ring)

Additional semantic tokens (from intake/index.html `:root` — include in chat.html verbatim):
- `--surface-strong: #fffdf9` — input backgrounds
- `--ink: #171717` — primary text
- `--muted: #6b7280` — secondary text, labels, stage indicator
- `--line: rgba(23,23,23,0.1)` — borders, dividers
- `--accent-soft: rgba(17,17,17,0.06)` — hover states, status pills
- `--success-bg: #eef6f1` / `--success-ink: #21593e` — success banners
- `--error-bg: #fbefef` / `--error-ink: #8b2d38` — error banners
- `--shadow: 0 24px 70px rgba(23,23,23,0.08)` — card elevation
- `--radius-xl: 30px`, `--radius-lg: 22px`, `--radius-md: 18px`

## Interaction States

### Input Composer
- **Default:** textarea enabled, send button enabled, border `var(--line)`
- **Focus:** border `rgba(17,17,17,0.26)`, box-shadow `0 0 0 4px rgba(17,17,17,0.06)`
- **Loading (awaiting API):** textarea disabled (`opacity: 0.64`), send button disabled (`cursor: wait`), loading bubble visible in thread
- **Empty (no text):** send button disabled

### Solution Option Cards
- **Default:** card visible with "เลือกตัวเลือกนี้" button, `cursor: pointer`
- **After selection:** all buttons in card strip set to `disabled`, `opacity: 0.64`, `cursor: default` (source: CONTEXT.md D-03c)

### Loading Bubble
- Shows animated typing-dots (3 spans, bounce animation from intake/index.html)
- Shows rotating stage label (text content only, color `var(--muted)`)
- Stage rotates every 8s via `setInterval` (source: CONTEXT.md D-04b)
- Removed on API success or error; `clearInterval` called in both paths

### Sidebar Project Items
- **Default:** project name + formatted date, `cursor: pointer`
- **Active/selected:** `background: var(--accent-soft)` highlight
- Click loads conversation history into thread (if endpoint available) or is display-only if deferred

### Download Button
- Appears only when `stage === 'complete'` in API response (source: CONTEXT.md D-05b)
- Renders as anchor `<a href="/api/proposals/:projectId/download" download>` styled as primary button

## Layout Contract

### Full-Height App Shell (chat/chat.html)

```
body [display:flex; height:100vh; overflow:hidden]
├── .sidebar [width:240px; flex-shrink:0; height:100vh; overflow-y:auto; border-right:1px solid var(--line)]
│   ├── .sidebar-header [padding:16px; border-bottom:1px solid var(--line)]
│   │   └── button.primary "New Chat"
│   └── .project-list [flex:1; overflow-y:auto; padding:8px]
│       └── .project-item* [padding:12px 16px; border-radius:var(--radius-md); cursor:pointer]
└── .main [flex:1; height:100vh; display:flex; flex-direction:column; overflow:hidden]
    ├── .thread [flex:1; overflow-y:auto; padding:20px 24px]
    │   └── .message* (.user | .assistant)
    │       ├── .message-label [font-size:13px; color:var(--muted)]
    │       └── .bubble (.user | .assistant)
    └── .composer [flex-shrink:0; border-top:1px solid var(--line); padding:16px 20px; background:var(--surface)]
        ├── textarea.prompt-box [min-height:72px; resize:vertical]
        └── .composer-row [display:flex; justify-content:space-between]
            ├── span.hint [color:var(--muted); font-size:13px]
            └── button.primary "ส่ง"
```

Critical CSS: `overflow:hidden` on both `body` and `.main` is mandatory. Without it, `.thread` bleeds past viewport and the composer scrolls off-screen.

### Login Page (login/login.html)

Single centered card, mirrors admin/index.html panel pattern:
- Card: `background:var(--surface); border:1px solid var(--line); border-radius:var(--radius-xl); padding:32px; max-width:400px; margin:auto`
- Page background: `var(--bg)`, full 100vh centered flex

## CDN Libraries

| Library | Version | CDN URL | Purpose |
|---------|---------|---------|---------|
| marked.js | 17.0.5 | `https://cdn.jsdelivr.net/npm/marked@17.0.5/lib/marked.umd.js` | Markdown → HTML for assistant messages |
| DOMPurify | 3.3.3 | `https://cdn.jsdelivr.net/npm/dompurify@3.3.3/dist/purify.min.js` | XSS sanitization before innerHTML |

Load order: marked.js first, DOMPurify second, then chat.js — all as plain `<script>` tags (not `type="module"`).
SRI hashes: obtain from jsDelivr "Copy with SRI" UI before committing. Add `integrity=` and `crossorigin="anonymous"` attributes.
API: use `marked.parse(text)` — NOT `marked(text)` (removed in v5+).
