  <action>
Append the following DDL to the end of supabase/schema.sql (after the match_knowledge_base function), per D-10/D-11/D-12:

```sql
create table if not exists conversations (
  id uuid primary key default gen_random_uuid(),
  project_id uuid not null references projects(id) on delete cascade,
  user_id uuid not null,
  stage text not null default 'greeting',
  created_at timestamptz not null default now(),
  updated_at timestamptz not null default now()
);

create table if not exists messages (
  id uuid primary key default gen_random_uuid(),
  conversation_id uuid not null references conversations(id) on delete cascade,
  role text not null check (role in ('user', 'assistant')),
  content text not null,
  created_at timestamptz not null default now()
);

create index if not exists idx_conversations_user_id on conversations(user_id);
create index if not exists idx_conversations_project_id on conversations(project_id);
create index if not exists idx_messages_conversation_id on messages(conversation_id);
```

Stage values per D-03: greeting, discovery, solution, awaiting_selection, bom, proposal, complete.
The stage column is text (not enum) for flexibility. Validation happens in application code.

Export the following functions:

1. `createConversation(projectId, userId)` — Insert into conversations table with stage='greeting'. Returns `{ saved: boolean, conversation: { id, project_id, user_id, stage, created_at, updated_at } }`. Local fallback returns `{ saved: false, conversation: { id: randomUUID(), project_id: projectId, user_id: userId, stage: 'greeting', created_at: new Date().toISOString(), updated_at: new Date().toISOString() } }`.

5. `getMessagesByConversation(conversationId)` — Select * from messages where conversation_id = conversationId order by created_at asc. Returns array of messages. Local fallback returns [].

Import: `import { randomUUID } from "node:crypto";` and `import { getSupabaseAdmin } from "./supabase.js";`
