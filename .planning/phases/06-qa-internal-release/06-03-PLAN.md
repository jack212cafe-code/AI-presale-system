  <action>
    Create `scripts/perf-check.js` per D-07 and D-08. This is a manual timing script that runs against a LIVE server (not mock mode). It must NOT set AI_PRESALE_FORCE_LOCAL=1 (per Pitfall 5 in research).

    The script should:

    1. Check that the server is running at `BASE_URL` (default `http://localhost:3000` or from env `PUBLIC_BASE_URL`)
    2. Verify live credentials are present (import `hasOpenAi` from `lib/config.js` — abort with message if not)
    3. Log in to get a session cookie (POST /api/auth/user-login with test credentials, or use admin auth)
    4. **Pipeline test (threshold: 60s):** POST /api/chat with a brief like "HCI + Backup for 100 users, 50 VMs, 20TB storage", measure wall-clock time from request to response
    5. **Per-turn test (threshold: 10s):** Use the conversation_id from step 4, POST /api/chat with "1" to select option, measure wall-clock time
    6. Print results table:
       ```
       === Performance Check Results ===
       Pipeline (turn 1):  {elapsed}ms / 60000ms threshold — {PASS|FAIL}
       Selection (turn 2): {elapsed}ms / 10000ms threshold — {PASS|FAIL}
       ```

    Script structure:
    ```javascript
    import { hasOpenAi } from "../lib/config.js";

    const BASE_URL = process.env.PUBLIC_BASE_URL || "http://localhost:3000";
    const PIPELINE_THRESHOLD = 60000;
    const TURN_THRESHOLD = 10000;

    if (!hasOpenAi()) {
      console.error("ERROR: OPENAI_API_KEY not set. perf-check requires live credentials.");
      process.exit(1);
    }

    // ... login, time turn 1, time turn 2, print results ...
    ```

    Use `fetch` (built-in Node 24) for HTTP calls. Login with admin credentials (POST /api/auth/login with ADMIN_PORTAL_PASSWORD from env) to get session cookie for subsequent requests.

    Include error handling: if server is unreachable, print clear message and exit 1.

    After creating the script, update `06-VALIDATION.md` frontmatter:
    - Set `nyquist_compliant: true`
    - Set `wave_0_complete: true`
    This signs the validation contract confirming Wave 0 gaps (perf-check.js) are now closed.
  </action>
