# Playbook Security Baseline for Infrastructure

- Security baseline recommendations should cover identity, privileged access, network segmentation, backup immutability, logging, patching, vulnerability visibility, and operational recovery.
- Avoid security as a bolt-on. Server, storage, network, virtualization, and DevOps recommendations should all describe how access control, change policy, and monitoring fit into daily operations.
- Minimum baseline assumptions should include MFA for privileged roles, centralized logging, tested backup recovery, version control for configuration changes, and software lifecycle ownership.
- Position additional controls based on business risk and operational maturity, not only on feature availability. A control that cannot be operated correctly becomes compliance theater.
- For presale proposals, document the boundary between the baseline included with the main solution and the extra controls that require a dedicated security workstream.
