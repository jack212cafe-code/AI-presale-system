# Pattern Secure Network Edge and Segmentation

- Use this pattern when the customer needs stronger segmentation, branch connectivity, SD-WAN policy control, or integrated edge security operations.
- Discovery focus: site topology, Internet breakout design, branch behavior, segmentation boundaries, identity integration, and logging or SOC expectations.
- Recommended architecture shape: resilient edge security platform with clear routing boundaries, segmentation policy, centralized management, and monitored enforcement points.
- Key design checks: throughput with services enabled, HA design, routing ownership, policy lifecycle, and operational rollback.
- Common risks: sizing from raw datasheet throughput, mixing network and security ownership ambiguously, and centralizing policy without change discipline.
- Commercial guidance: separate baseline enforcement, SD-WAN capability, logging retention, and advanced inspection into understandable phases.
