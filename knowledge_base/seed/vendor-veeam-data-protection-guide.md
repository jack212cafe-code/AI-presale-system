# Vendor Veeam Data Protection Guide

- Position Veeam when the customer needs strong VM and workload protection, flexible recovery options, immutability, and operationally mature backup governance across on-premises and hybrid environments.
- Presale discovery must capture recovery objectives, repository design, proxy placement, change rate, retention, immutability requirements, WAN limits, and restore testing expectations before sizing.
- Do not size only on front-end protected capacity. Repository growth, synthetic full behavior, backup window pressure, and recovery concurrency all affect the architecture materially.
- Separate backup success from recoverability. The design should include verification, restore drills, role separation, credential protection, and ransomware-aware recovery operations.
- When presenting Veeam solutions, document dependencies on storage targets, hardened repositories, object storage policy, and operational ownership of backup monitoring.
