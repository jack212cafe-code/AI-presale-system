# Vendor Fortinet Security Edge Guide

- Position Fortinet when the customer needs consolidated security controls across firewalling, SD-WAN, segmentation, and edge visibility with a preference for centralized policy management.
- Presale discovery should capture Internet breakout model, branch behavior, east-west segmentation, VPN requirements, identity dependencies, log retention, and security operations maturity.
- Security architecture must be sized around throughput with real services enabled. Raw datasheet throughput is not a valid design basis once SSL inspection, IPS, logging, or segmentation policy are introduced.
- Explain how routing, switching, security, and monitoring responsibilities meet at the enforcement points. Many presale failures happen because ownership and operational assumptions stay implicit.
- Include policy lifecycle, log review, HA behavior, firmware strategy, and rollback process in the proposal so the design remains operable after go-live.
